# CLAUDE.md — Bicycle Shop App

## Build & Dev

- `npx vite build` — production build (~3s, outputs to `dist/`)
- Dev server: use `preview_start` with `.claude/launch.json` (Vite on port 5173, `--host 0.0.0.0`)
- launch.json: use `node` + `node_modules/vite/bin/vite.js` — `npx` is not resolved by preview_start
- No test suite — verify changes via preview server + build check

## Code Conventions

- React 19 SPA, no router — page routing via `currentPage` state in App.jsx
- Shared UI primitives in `src/components/ui.jsx` (Card, Badge, StatCard, EmptyState, etc.)
- Tailwind CSS 4 via `@tailwindcss/vite` plugin — no `tailwind.config.js`
- Icons: `lucide-react` — always verify imports exist before adding new ones
- IDs generated via `genId()` = `"id_" + Date.now() + "_" + random`
- StatCard `bgMap` must include every color passed by consumers (e.g. `teal` for Trip Prep)
- Role-based access: 4 roles (owner/manager/tech/client), permissions in `src/lib/constants.js`

## Data Layer

- Offline-first: `loadDB()` (sync, localStorage) → `fetchDB()` (async, Supabase) → `saveDB()` (both)
- Snake_case (Postgres) ↔ camelCase (JS) via `snakeToCamel`/`camelToSnake` in `src/lib/db.js`
- `SKIP_KEYS`: `_schemaVersion`, `_legacyMaintenance`, `_autoGenerated`, `_bookingId`
- `saveDB` checks auth session before Supabase sync to avoid RLS errors

## Hosting

- GitHub: `github.com/JaredTheHammer/bicycle-shop-app` (public, `master` branch)
- Vercel: `bicycle-shop-app.vercel.app` — auto-deploys on push to master
- Supabase anon key is publishable (client-side by design, protected by RLS)

## Gotchas

- PRs require a feature branch — create branch before committing, not after
- Main branch is `master` (not `main`)
- Main content scrolls via nested `flex-1 overflow-y-auto` div, not `window` — `window.scrollTo()` won't work
- Sidebar: hamburger menu appears at `max-lg:` (≤1024px), full sidebar at `lg:`
- Vite chunk size warning is expected — single-page app bundles everything
- `syncQueue.js` exists but is dormant — doesn't match full-collection-diff architecture
